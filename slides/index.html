The experiment

One yellow iPhone. The same prompts, run twice.

  • Fable 5 and Opus, in parallel folders, same references and skills.
  • Every level's HTML frozen the moment it was built — SHA-256 checksummed in the repo.

Ground rules

Treat prompting experiments like experiments.

  • Verbatim evidence — no touch-ups after the fact.
  • Named output files per level, so history stays immutable.
  • One variable changed at a time (even: "do NOT use superpowers plugin").

Level 1 · Grab and go

The prompt everyone writes

Claude, I want you to build for me a simple website or just a nice looking website that is cool about iPhones that are yellow. Build me one of those website. name the file as yellow-iphone-web-level-1.html

Show, don't tell.

Level 3 · Result

The model catches its own bug

  • File size doubled, 20KB → 56KB: that's the design system arriving.
  • Fable's WCAG pass caught a real CSS specificity bug — white-on-yellow nav — and fixed it to 11.3:1 contrast.
  • Let the model verify: dark mode, 320px reflow, contrast. It finds real bugs.

Level 4 · UI snipping

Don't re-engineer the fireplace — copy it.

  • Production components from 21st.dev, Magic UI, CodePen, Mobbin.
  • Paste the code — or just screenshots of it — and ask Claude to integrate.
  • Here: balloons-js, a celebration animation, into a no-build HTML page.

Level 6 · Data research

A beautiful site that doesn't convert is a Ferrari with no engine.

  • Firecrawl scrapes live sites: type scale, palette, spacing, tone — from apple.com itself.
  • Bigger pattern: research winners vs losers in your market, score them, build from evidence.
  • The actual scrapes are preserved in the repo.
Firecrawl

Level 6 · The overwrite incident

Two sessions raced. One file lost.

  • The level-7 session silently clobbered level-6.html while it was being built.
  • Fable diagnosed the race, mined the other session's transcript for the build recipe, and restored the file — flagged as a rebuild, not the original.
  • Lessons: partition concurrent sessions; transcripts are recoverable state.

Level 7 · Design extraction

The strongest lever

  • A skill extracts a site's full design language: tokens, type ladder, easing, load-bearing rules.
  • Claude rebuilds your product in that language. Translation, not cloning.
  • Target here: antigravity.google — grey palette, weight-450 headings, pill shapes, alpha borders.
antigravity.google — the source site design-language skill (in this repo)

Meta-lessons

Four inequalities to remember

  • References > adjectives
  • Skills > instructions
  • Research > invention
  • Verification > hope

Tricks & gotchas

Field notes

  • Name output files — keep every level as immutable evidence.
  • Change one variable at a time; exclude plugins explicitly.
  • Partition concurrent sessions — they will race on files.
  • Budget one extra prompt from blueprint → deliverable.
  • Let the model look at its own screenshots.

What this means for Kite

The scaffolding is the product

  • Agent-built UI is production-viable today — with skills, references, data access, and paid tools (Firecrawl, OpenArt).
  • Every level up the ladder, the agent consumed more external services autonomously.
  • Agents that research, buy assets, and verify on their own need identity and payments — that's Agent Passport.
