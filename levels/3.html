Level 3 of 7

Design skills

Install packaged design expertise so the model knows what good looks like — then it stops eating crayons and starts playing the piano.

The technique

Skills are prepackaged instruction sets written by people who obsess about design: UI styles, color palettes, font pairings, UX rules. Point Claude at a skill repo (ui-ux-pro-max, power-design, shadcn) and it internalizes the rulebook before writing a line of CSS.

This level combined two skills with a real product photo of the phone. The result roughly doubled in size versus level 2 — that's where the design system went.

The prompts that did it

Invoke every relevant design skill
Hey there. I'd like to use all the best skills and the design principles to build me a beautiful website selling yellow iPhones. Make sure that's using all relevant skills that I have given you for building beautiful design @reference-real-iphone.png https://github.com/ItsssssJack/power-design.git name the file as yellow-iphone-web-level-3.html

Fable invoked the ui-ux-pro-max skill plus power-design brand references, then ran a full QA pass: dark mode, 320px reflow, a WCAG contrast check that caught a real CSS specificity bug and fixed it to 11.3:1 contrast. The Opus prompt also added ui-ux-pro-max explicitly and pinned the variables: “do NOT use superpowers plugin in this session.”

Cost & time

What changed vs level 2

  • A coherent design system appears: consistent spacing scale, type hierarchy, real color palette.
  • File size roughly doubles — the craft is in the CSS.
  • The model starts catching its own design bugs when allowed to verify.

What this level taught

  • Skills > instructions: packaged expertise compounds with raw model ability.
  • Let the model verify (contrast, viewports) — it finds real bugs.
  • Control your variables when comparing models: exclude plugins explicitly.
